# チューニングガイド

GRID_SIZES の調整方法、印刷プレビューでの確認手順、PDF の種類ごとの注意点を説明します。

---

## GRID_SIZES の調整

### 調整対象のファイル

`src/core/grid_config.py` の `GRID_SIZES` 定数を編集します。

### 調整可能なパラメータ

| パラメータ | 影響 | 調整指針 |
|----------|------|---------|
| `max_cols` / `max_rows` | グリッドの密度（列数・行数） | 増やす → 高精度だが空セル増加。減らす → 粗いが重なりにくい |
| `excel_col_width` | Excel 列幅 | `max_cols` 変更時は印刷範囲に合うよう再計算が必要 |
| `excel_row_height` | Excel 行高 | `max_rows` 変更時に同様 |
| `default_font_size` | デフォルトフォントサイズ | セルサイズに合わせて調整。1pt: 7pt、2pt: 6pt |
| `margin_*` | 印刷マージン（インチ） | Excel の印刷設定に反映 |

### max_cols / max_rows の調整手順

1. `GRID_SIZES` の値を変更
2. `auto` コマンドで対象 PDF を再変換
3. 生成された Excel を開き、印刷プレビューで確認
4. 問題があれば値を調整して再実行

### セルサイズの計算方法

`compute_grid_coords()`（`src/core/grid.py`） でコンテンツ領域を均等分割するため：

```
1列の幅 = コンテンツ幅 / max_cols
1行の高さ = コンテンツ高さ / max_rows
```

`max_cols` を増やすとセル幅が狭くなり、より細かいレイアウトを表現できますが、テキストがセルに収まりにくくなります。

## 印刷プレビューでの確認手順

### 確認すべきポイント

1. **テキストの切れ・重なり**: セルに文字が収まっているか
2. **罫線の位置**: PDF 原本と比較して罫線がずれていないか
3. **空白の過不足**: 不要な空行・空列がないか
4. **印刷範囲**: 1ページに収まっているか

### 確認手順

1. 生成された `.xlsx` を Excel で開く
2. 「ファイル」→「印刷」で印刷プレビューを表示
3. 出力先にコピーされた元 PDF と見比べる
4. 罫線プレビュー画像（`prompts/` 内の `_excel_page*.png`）も参考にする

### 1pt と 2pt の使い分け

| サイズ | 特徴 | 向いているPDF |
|-------|------|-------------|
| 1pt | 高密度グリッド（列幅 3.48mm） | 列数が多い表、細かいレイアウト |
| 2pt | 中密度グリッド（列幅 6.18mm） | シンプルな表、テキスト中心のレイアウト |

`auto` コマンドは常に両方を出力するため、比較して適切な方を選んでください。

## PDF の種類ごとの注意点

### テーブルが多い PDF

- pdfplumber のテーブル検出が主に機能する
- テーブル内テキストは word 座標ベースで精密に配置される
- 結合セルは自動認識されるが、複雑な結合パターンではズレが生じることがある

### テキスト中心の PDF（請求書・帳票）

- テーブル外のテキストは word の `_row` / `_col` でグループ化して配置
- 水平ギャップ分割 (`split_by_horizontal_gap()`（`src/utils/text.py`）) が同一行内のテキストブロックを分離
- テキスト量が多い場合は 2pt の方が読みやすい出力になる傾向

### 罫線が多い PDF

- `rects` として検出され、形状に応じて水平線・垂直線・矩形に分類される
- ページ外枠フィルタ（面積80%以上 × セル数4以下）で誤検出を防止
- 装飾的な罫線が多い場合、`correct` コマンドで `remove_border` を使って不要な罫線を除去

### スキャン PDF（画像のみ）

- `check` コマンドで事前に判定可能
- テキストが抽出できないため `auto` コマンドでは変換不可
- 事前に OCR 処理が必要

## トラブルシューティング

### テキストが重なる

- `max_cols` / `max_rows` を増やしてグリッドを細かくする
- ただし空セルも増えるためバランスを見る

### 空セルが多すぎる

- `max_cols` / `max_rows` を減らす
- コンテンツ密度が低い PDF では自然な結果

### テーブルが検出されない

- `src/parser/pdf_extractor.py` の `_table_settings` を確認
- `snap_tolerance` / `edge_min_length` の値が大きすぎると検出漏れの可能性
- PDF の罫線が画像として埋め込まれている場合は検出不可

### 外枠がテーブルとして検出される

- `pdf_extractor.py` の外枠除外ロジック（面積80%閾値）を確認
- 閾値を調整する場合は、正当なテーブルが除外されないよう注意

### correct で罫線が削除されない

- `remove_border` は完全包含判定のため、指定範囲が対象罫線を完全に含む必要がある
- 指定範囲を対象罫線より少し広くする
