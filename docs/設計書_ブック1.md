## Sheet 3: ツール概要

### ■ ツールの目的

PDFを解析し、任意のPDFレイアウトを維持したままA4/A3方眼Excelに変換するツール。テーブルの列ズレのない高精度な方眼紙レイアウトへの変換を、LLMを最小限に抑えた完全自動パイプラインで実現する。

---

### ■ ファイル配置

| 項目 | パス | 補足 |
|-----|------|------|
| 入力PDFの配置先 | `data/in/<任意のフォルダ名>/<ファイル名>.pdf` | 例: `data/in/test1/sample.pdf` |
| 出力先（自動生成） | `data/out/<フォルダ名>/` | 例: `data/out/test1/` |
| check結果の出力先 | `data/doc/pdflist_check.csv` | checkコマンド実行後に生成 |

---

### ■ 処理パイプライン

| Step | 処理 | 関数 / ファイル | 説明 |
|------|------|--------------|------|
| 1 | PDF → データ抽出 | `extract_pdf_data()`<br>`src/parser/pdf_extractor.py` | words / table_cells / rects / edges を抽出 |
| 2 | グリッドパラメータ設定 | `setup_grid_params()`<br>`src/core/grid.py` | 用紙サイズ（A4/A3）・向き（縦/横）を検出 |
| 3 | 座標変換 | `compute_grid_coords()`<br>`src/core/grid.py` | PDF座標 → Excelグリッド座標に変換 |
| 4 | レイアウトJSON生成 | `generate_layout()`<br>`src/core/layout.py` | テキスト要素・罫線要素を収集 |
| 5 | Excel描画 | `render_layout_to_xlsx()`<br>`src/renderer/excel.py` | openpyxlで .xlsx を生成 |
| 6 | プレビュー生成 | `generate_border_preview()`<br>`src/renderer/preview.py` | 罫線プレビュー画像を生成（視覚的検証用） |
| 7 | 差分オーバーレイ生成 | `generate_diff_overlay()`<br>`src/renderer/preview.py` | PDF原本に現プレビュー罫線を半透明赤で重ねた差分画像を生成（LLM向け） |

## Sheet 4: コマンドリファレンス

### ■ auto：PDF → Excel 自動変換

**実行コマンド**

```
python -m src.main auto
python -m src.main auto --pdf data/in/test1/sample.pdf
```

| 項目 | 内容 |
|-----|------|
| 処理対象 | `data/in/` 配下の全PDF（または `--pdf` で指定した1ファイル） |
| 出力サイズ | 1pt・2pt の2サイズを同時出力 |
| 出力先 | `data/out/<relative_path>/`（`data/in/` からの相対パスで決まる） |
| 同時生成物 | レイアウトJSON・グリッドパラメータJSON・PDFコピー・プレビュー画像・検証プロンプト |
| 用紙検出 | A4/A3・縦/横 をPDFの寸法から自動判定 |

**グリッドサイズ仕様**

| グリッド | 列幅(mm) | 行高(mm) | A4縦(cols×rows) | A4横(cols×rows) | フォント |
|--------|--------|--------|--------------|--------------|--------|
| 1pt (A4) | 3.48 | 6.44 | 58×44 | 85×30 | 7pt MS 明朝 |
| 2pt (A4) | 6.18 | 6.44 | 35×44 | 52×30 | 6pt MS 明朝 |
| 1pt (A3) | 3.48 | 6.44 | 88×64 | 125×45 | 7pt MS 明朝 |
| 2pt (A3) | 6.18 | 6.44 | 51×64 | 72×45 | 6pt MS 明朝 |

---

### ■ check：PDF種別判定

**実行コマンド**

```
python -m src.main check
```

| 判定・項目 | 説明 |
|---------|------|
| 出力先 | `data/doc/pdflist_check.csv` |
| 通常PDF（テキストあり） | 少なくとも1ページにテキストが含まれる → autoコマンドで変換可能 |
| スキャンPDF（画像） | 全ページにテキストがない（画像のみ）→ 事前にOCR処理が必要 |
| エラー | ファイルが破損している等で読み取り不可 |

---

### ■ correct：視覚的検証の修正を適用

**実行コマンド**

```
python -m src.main correct --pdf data/in/test1/sample.pdf   ← 推奨
python -m src.main correct                                   ← 全対象一括
```

| 項目 | 内容 |
|-----|------|
| 前提 | autoコマンド実行後、フェーズ1・フェーズ2のプロンプトでLLMに修正指示を作成していること |
| 処理内容 | phase1・phase2の修正指示JSONをマージし、layout JSONを更新してExcelを再生成する |
| 修正アクション: remove_edges | フェーズ1: 指定IDの罫線を削除する |
| 修正アクション: add_edge | フェーズ2: 指定位置に罫線を追加する（inclusive座標） |
| 推奨 | 個別PDFを `--pdf` で指定する（全件一括は対象が多い場合に注意） |

**フェーズ1: `_phase1_corrections_page{N}.json` の形式**

```json
{
  "corrections": [
    {"action": "remove_edges", "page": 1, "ids": [3, 17, 42]}
  ]
}
```

**フェーズ2: `_phase2_corrections_page{N}.json` の形式（座標は inclusive）**

```json
{
  "corrections": [
    {"action": "add_edge", "page": 1, "type": "H", "row": 5, "col_start": 3, "col_end": 12},
    {"action": "add_edge", "page": 1, "type": "V", "col": 3, "row_start": 1, "row_end": 8}
  ]
}
```

> `col_end` / `row_end` は最終セルの番号（inclusive）。col 3〜12 なら `col_end=12`（13 ではない）。

## Sheet 5: ソース参照ガイド（開発者向け）

### ■ カテゴリ1: グリッド・用紙設定

**ファイル:** `src/core/grid_config.py`　**定数:** `GRID_SIZES`

| やりたいこと | 変更するキー | 対象グリッド | 備考 |
|------------|-----------|-----------|------|
| セルの列幅を変えたい | `col_width_mm` / `excel_col_width` | 1pt / 2pt / 1pt_a3 / 2pt_a3 | mm値とExcel内部値の両方を変更する |
| セルの行高を変えたい | `row_height_mm` / `excel_row_height` | 全グリッド共通 | |
| A4縦の列数・行数を変えたい | `max_cols` / `max_rows` | 対象グリッド | |
| A4横の列数・行数を変えたい | `max_cols_landscape` / `max_rows_landscape` | 対象グリッド | |
| 印刷マージンを変えたい | `margin_left/right/top/bottom` | 対象グリッド | 単位: cm |
| デフォルトフォントを変えたい | `font_name` / `default_font_size` | 対象グリッド | 例: MS 明朝 → MS ゴシック |

---

### ■ カテゴリ2: 罫線スタイルの閾値

**ファイル:** `src/core/constants.py` / `src/utils/font.py`（`linewidth_to_border_style()`）

| 定数名 | デフォルト値 | 意味 | 変更効果 |
|------|-----------|------|--------|
| `BORDER_THIN_MAX_LW` | 1.0 | linewidth ≦ この値 → thin 罫線 | 値を上げるとthin判定が増える |
| `BORDER_MEDIUM_MAX_LW` | 2.0 | linewidth ≦ この値 → medium 罫線（超えるとthick） | 値を上げるとmedium判定が増える |

---

### ■ カテゴリ3: テキスト・座標の許容誤差

**ファイル:** `src/core/constants.py`

| 定数名 | デフォルト値 | 意味 | 変更効果 |
|------|-----------|------|--------|
| `WORD_IN_BBOX_TOL` | 2.0 | テキストがbbox内かの判定誤差（PDF point） | 小さくするとテキスト欠落リスク減・誤配置リスク増 |
| `WORD_IN_TABLE_TOL` | 2.0 | テーブル内外のテキスト判定誤差 | 小さくするとテーブル外テキスト漏れが減る |
| `EDGE_MERGE_GAP_TOL` | 2.0 | エッジ連結の許容誤差 | 小さくすると罫線が細切れになりやすい |
| `VISUAL_LINE_GAP` | 3.0 | 視覚行のtop差分閾値（行分割の感度） | 小さくすると行が細かく分割される |
| `HORIZONTAL_GAP_FACTOR` | 2.0 | 水平ギャップのfont_size倍率（テキスト分割） | 大きくするとテキストが結合されやすくなる |

---

### ■ カテゴリ4: ページレイアウト構造

**ファイル:** `src/core/constants.py`

| 定数名 | デフォルト値 | 意味 | 変更効果 |
|------|-----------|------|--------|
| `COL_OFFSET` | 1 | 印刷範囲の左に空ける列数 | 0にすると左端から印刷が始まる |
| `ROW_PADDING` | 1 | ページ間に挿入する空行数 | 0にするとページ間に隙間がなくなる |

---

### ■ カテゴリ5: 視覚的検証プロンプト

**ファイル:** `src/templates/prompts.py`　**定数:** `VISUAL_PHASE1_PROMPT` / `VISUAL_PHASE2_PROMPT`

フェーズ1（削除判定）・フェーズ2（追加判定）用のプロンプトテンプレート。判定基準・出力形式・座標の規約を記述している。修正したい場合はそれぞれの定数を直接編集する。

---

### ■ 変更影響範囲マップ

| 変更箇所 | 直接影響するファイル | 影響の内容 |
|---------|----------------|---------|
| `GRID_SIZES`（grid_config.py） | grid.py / excel.py / preview.py / auto_layout_service.py | グリッド計算・Excel描画・プレビュー画像・プロンプト整形すべてに波及 |
| `BORDER_THIN/MEDIUM_MAX_LW`（constants.py） | font.py → excel.py | 全罫線のthin/medium/thick判定が変わる |
| `WORD_IN_BBOX_TOL` / `WORD_IN_TABLE_TOL`（constants.py） | layout.py | テキスト要素の収集結果が変わる（欠落・誤配置） |
| `EDGE_MERGE_GAP_TOL`（constants.py） | pdf_extractor.py → grid.py → layout.py | 罫線の連結結果が変わり、レイアウト全体に影響 |
| `VISUAL_LINE_GAP` / `HORIZONTAL_GAP_FACTOR`（constants.py） | layout.py / text.py | テキストの行分割・水平分割結果が変わる |
| `COL_OFFSET` / `ROW_PADDING`（constants.py） | excel.py | Excel上の印刷範囲・ページ間レイアウトが変わる |
| `VISUAL_PHASE1/2_PROMPT`（templates/prompts.py） | auto_layout_service.py | correctコマンド用のフェーズ1/2プロンプト出力が変わる |
