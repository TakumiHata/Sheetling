# プロジェクト指示書

## ロール
あなたは、複雑なドキュメント解析とデータ構造化のエキスパートエンジニアです。

## プロジェクト概要
Accessから出力された帳票PDFを、元の見た目（方眼レイアウト・色彩）を完全に維持したままExcel（.xlsx）に自動変換するシステムの開発。

## プロジェクトの背景と最終目的
- **ビジネス課題**: ユーザーはAccessのレポート機能で生成された多様な帳票を、見た目（レイアウト、セル結合、色など）を維持したまま「方眼Excel」として出力したいと考えている。
- **技術的制約**: 
    - Accessから直接Excelへレイアウトを維持してエクスポートする標準機能は存在しない。
    - VBA等でのスクラッチ開発は保守性の観点から避けたい。
    - 出力される帳票フォーマットは複数存在し、事前定義が難しい（動的に変わる）。
- **最終目的**: 既存の高度なPDF変換ソフト（ABBYY FineReaderなど）を代替するような、「PDFの視覚情報を動的に解析して、未知のレイアウトであっても自動で方眼Excelを生成する内製パイプライン」を構築すること。
    - LLMにExcel生成してもらいます。必要なプロンプトとテキスト及びレイアウト構造の、mdファイルやjsonファイルの出力を担当するのがこのリポジトリの役目です。

## 要件（アーキテクチャ方針）
1. **方眼Excelレイアウト**: 
    - 出力先Excelを微小な正方形のセル（方眼紙）の集合として扱う。
    - PDF内の各要素の座標情報を基に、適切なセル範囲を結合・着色してレイアウトを再現する。
2. **動的解析**: 
    - テンプレート定義に頼らず、PDFの解析結果（Docling等を使用）から動的に構造を特定する。
3. **A4出力最適化**:
    - 生成されるExcelは、印刷時に**A4サイズ1ページ（または複数ページ）の幅に収まる**ように構成する。
    - 適切な余白調整（上下左右）を自動的に行い、印刷・PDF化が容易な設定をデフォルトとする。
4. **保守性**: 
    - Pythonをベースとした現代的なスタックを使用し、メンテナンスしやすい構成とする。

## 技術的アドバイス・ベストプラクティス
1. **ハイブリッド解析の活用**:
    - **Docling**: レイアウト構造（表、セクション）の抽出に使用。
    - **pdfplumber**: 色彩情報（背景色、罫線）および幾何学座標の抽出に使用。
    - **MarkItDown**: テキスト情報の精密な抽出とMarkdown化に使用。
2. **座標の正規化とグリッド化**:
    - PDFのポイント座標を直接Excelのセルにマッピングするのではなく、最小公約数的なグリッド単位（例：2mm単位や一定のpt数）を算出し、正規化（スナップ）させることでレイアウトを安定させる。
3. **LLMの役割の限定**:
    - LLMにはバイナリ生成ではなく、抽出された構造データに基づいた「Excel設計図（セル番地、結合、色、内容の命令セット）」を書かせることに集中させる。
